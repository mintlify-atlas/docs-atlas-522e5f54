This guide walks you through installing Codeplain and running the bundled `hello_world_python` example. By the end you will have a generated Python application sitting in a `build/` directory.

<Note>
  You need a Codeplain API key to generate code. Contact [support@codeplain.ai](mailto:support@codeplain.ai) to get one, or sign up at [platform.codeplain.ai](https://platform.codeplain.ai).
</Note>

## Prerequisites

- Python 3.11 (exactly — see [Installation](/installation) for details)
- A Codeplain API key
- macOS or Linux (Windows users: use WSL)

## Step 1 — Install Codeplain

The fastest way to install is with `uv`:

```bash
uv tool install codeplain
```

Alternatively, install with `pip`:

```bash
pip install codeplain
```

For a full breakdown of installation options, see [Installation](/installation).

## Step 2 — Set your API key

Export your API key as an environment variable:

```bash
export CODEPLAIN_API_KEY="your_api_key_here"
```

To persist it across sessions, add that line to your `~/.bashrc` or `~/.zshrc`.

## Step 3 — Clone the repository and navigate to the example

```bash
git clone https://github.com/Codeplain-ai/plain2code_client.git
cd plain2code_client/examples/example_hello_world_python
```

This directory contains:

```
example_hello_world_python/
├── hello_world_python.plain   # The spec file
├── config.yaml                # CLI configuration
├── harness_tests/             # Test harness
└── run.sh                     # Convenience run script
```

## Step 4 — Review the spec

Open `hello_world_python.plain` to see what you are about to render:

```plain
---
description: '"hello, world" in Plain (Python version)'
import:
  - python-console-app-template
---

***implementation reqs***

- :MainExecutableFile: of :App: should be called "hello_world.py".

***functional specs***

- Display "hello, world"

  ***acceptance tests***

  - :App: should exit with status code 0 indicating successful execution.
  - :App: should complete execution in under 1 second.
```

This spec imports the `python-console-app-template` (which defines the base structure of a Python console application), names the output file `hello_world.py`, and has one functional requirement: display `"hello, world"` — with two acceptance tests to verify it.

## Step 5 — Run codeplain

From inside the example directory:

```bash
codeplain hello_world_python.plain
```

Codeplain opens a terminal UI that shows rendering progress. The tool will:

1. Connect to the Codeplain API and validate your key
2. Parse the `.plain` file and resolve imported templates
3. Generate implementation code for each functional requirement
4. Run unit tests and acceptance tests against the generated code
5. Write the final output to `plain_modules/`

<Note>
  Rendering typically takes a few minutes. The API is doing real LLM-based code generation and test validation on the server side.
</Note>

If you prefer no TUI (for scripting or CI), pass `--headless`:

```bash
codeplain hello_world_python.plain --headless
```

## Step 6 — Run the generated code

Once rendering completes, a Python file has been generated. The example's `config.yaml` is configured with `copy-build: true`, so the final output is also copied to a `build/` folder:

```bash
cd build
python hello_world.py
```

Expected output:

```
hello, world
```

The process exits with status code 0.

## What just happened?

- Your `.plain` spec was sent to the Codeplain API.
- The API used large language models to generate a Python implementation that satisfies the spec.
- The generated code was tested against the acceptance tests defined in the spec.
- The verified output was written to `plain_modules/` (and copied to `build/`).

The key point: you did not write any Python. You wrote a specification, and Codeplain generated code that provably satisfies it.

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `Missing API key` | `CODEPLAIN_API_KEY` not set | Export the env var or pass `--api-key` |
| `Invalid API key` | Key is incorrect or expired | Check your key at [platform.codeplain.ai](https://platform.codeplain.ai) |
| `File not found` | Wrong working directory | Run from inside the example directory |
| `Connection error` | No internet or API unreachable | Check your network connection |
